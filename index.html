Creatable

Elegant HTML generation. No templates. Just Javascript.

Downloadcreatable.js

View the Project on GitHub

Creatable let's you build HTML in Javascript without templates.

['#content p a.small.button', { href: '/checkout' }, 'Checkout']
→
<div id="content">
  <p>
    <a class="small button" href="/checkout">Checkout</a>
  </p>
</div>

Why?

  • Switching back-and-forth between Javascript and a templating language is a pain.
  • Client-side templating is too heavy for simple markup.
  • You can decompose and reuse markup with plain ol' functions.
  • The nested array structure reflects the natural structure of HTML for readability.
  • Performing logic is easy and clean since it's Javascript all the way down.

Getting Started

Pass a simple array that represents an element to Creatable.create():

Creatable.create(["a", { href: "http://google.com" }, "Google"])

Creatable.create() returns a native DOM node. Append it to your document:

var node = Creatable.create(["a", { href: "http://google.com" }, "Google"]);
document.body.appendChild(node);

Attributes are optional. Pass an object of attributes as the second item in the array.

Creatable.create(['input', { type: 'input', name: 'city', value: 'Boulder' }])

You can easily nest child elements:

Creatable.create(['ul', [
  ['li', 'Item 1'],
  ['li', 'Item 2'],
  ['li', 'Item 3']
]])

The unique use of nested arrays results in a very intuitive structure that reflects the appearance of HTML:

var c = ["#content", [
  ["h1.prominent", "Blogs"],
  ["ul", [
    ["li a", { href: "http://functionsource.com" }, "FunctionSource"],
    ["li a", { href: "http://javascriptweblog.wordpress.com" }, "Javascript, Javascript"],
    ["li a", { href: "http://ejohn.org/category/blog" }, "John Resig"]
  ]]
]];
document.body.appendChild(Creatable.create(c));

Result:

<div id="content">
  <h1 class="prominent">Blogs</h1>
  <ul>
    <li><a href="http://functionsource.com">FunctionSource</a></li>
    <li><a href="http://javascriptweblog.wordpress.com">Javascript, Javascript</a></li>
    <li><a href="http://ejohn.org/category/blog">John Resig</a></li>
  </ul>
</div>

Documentation

Specify ids and classes like in CSS

Creatable.create(["div#footer", [
  ["p.fine-print", "Don't forget to read this!"]
]])
<div id="footer">
  <p class="fine-print">Don't forget to read this!</p>
</div>

HTML is automatically escaped

Creatable.create(['p', 'Something <strong>important</strong> to say.'])
<p>Something &lt;strong&gt;important&lt;/strong&gt; to say.</p>

You can unescape HTML by adding { html: true }

Creatable.create(['p', { html: true }, 'Something <strong>important</strong> to say.'])
<p>Something <strong>important</strong> to say.</p>

Document Fragments

Creatable.create([[
  ["p", "First paragraph!!!"],
  ["p", "Second paragraph!!!"],
  ["p", "Third paragraph I'm bored"]
]]);
<p>First paragraph!!!"</p>
<p>Second paragraph!!!"</p>
<p>Third paragraph I'm bored</p>

Works great with underscore

var links = [
    { url: "http://functionsource.com", label: "FunctionSource" },
    { url: "http://javascriptweblog.wordpress.com", label: "Javascript, Javascript" },
    { url: "http://ejohn.org/category/blog", label: "John Resig" }
];

var buildLinkItem = function(model) {
    return ["li", [
        ["a", { href: model.url }, model.label]
    ]];
};

Creatable.create(["ul", _.map(links, buildLinkItem)])
<ul>
    <li><a href="http://functionsource.com">FunctionSource</a></li>
    <li><a href="http://javascriptweblog.wordpress.com">Javascript, Javascript</a></li>
    <li><a href="http://ejohn.org/category/blog">John Resig</a></li>
</ul>

Built-in support for jQuery elements

Creatable.create(["#myModule", [
    ["h1", "This is a header"],
    "Some text.",
    $("<div><p>Go, Go jQuery</p></div>")
]])
<div id="myModule">
    <h1>This is a header</h1>
    Some text.
    <div><p>Go, Go jQuery</p></div>
</div>

Render a string of HTML instead of a DOM node

var html = Creatable.createHtml(['a#go.small.button', 'test']);
console.log(html); //<a id="go" class="small button">test</a>
